# Додаток «Особиста бібліотека» (Book Library)

Цей проект — це мобільний додаток для управління власною колекцією книг, що дозволяє користувачам шукати видання через публічні API та вести облік прочитаної літератури.

## Функціональні можливості

### 1. Авторизація та безпека
*   **Система облікових записів:** Локальна реєстрація та вхід користувачів.
*   **Управління сесіями:** Збереження стану авторизації (JWT-подібна логіка) для автоматичного входу при повторному відкритті додатка.

### 2. Каталог та пошук (Open Library API)
*   **Глобальний пошук:** Пошук книг за назвою або автором через інтеграцію з Open Library API.
*   **Детальна інформація:** Перегляд детальних даних про книгу, включаючи:
    *   Обкладинку.
    *   Повний опис якщо доступний.
    *   Авторів та рік першого видання.

### 3. Особиста бібліотека
*   **Статуси книг:** Можливість додавати книги до категорій «Прочитано» або «Хочу прочитати».
*   **Управління даними:** Видалення книг із власної колекції.
*   **Особисті нотатки:** Додавання власної оцінки від 1 до 5 зірок та текстових заміток до кожної книги у бібліотеці.

### 4. Робота в режимі Offline та Кешування
*   **Локальне сховище:** Повна підтримка перегляду своєї бібліотеки без доступу до інтернету.
*   **Кешування зображень:** Обкладинки книг зберігаються локально після першого завантаження.

## Технологічний стек

При розробці додатка було використано сучасні інструменти та бібліотеки екосистеми Flutter:

### Core (Ядро)
*   **Flutter SDK:** Основа додатка для створення кросплатформеного інтерфейсу.
*   **Provider:** Використовується як архітектурний паттерн для управління станом (State Management) та забезпечення реактивності UI.

### Data Storage (Збереження даних)
*   **sqflite (SQLite):** Надійна реляційна база даних для збереження інформації про користувачів та їхні особисті списки книг на пристрої.
*   **shared_preferences:** Для збереження легковагових даних, таких як ID поточного користувача та статус сесії.

### Networking (Мережа)
*   **http:** Бібліотека для виконання запитів до Open Library API та отримання даних у форматі JSON.
*   **cached_network_image:** Інструмент для ефективного завантаження, відображення та кешування зображень із мережі.

### Architecture (Архітектура)
Проект побудований за багатошаровою структурою для полегшення підтримки:
*   **Models:** Опис об'єктів даних.
*   **Services:** Логіка взаємодії з базою даних, API та аутентифікацією.
*   **Providers:** Проміжний шар, що пов'язує дані із користувацьким інтерфейсом.
*   **Screens:** Набір UI-компонентів та екранів додатка.
